# 手动测试清单（增量）- 别名、效果文案、历史互通

> 这份清单只覆盖本轮修复可能影响的区域，优先级从高到低。

## 1) 对抗别名显示（最高优先级）

- [ ] 打开对抗面板，发起方名称输入 `千早爱音,爱音`，对抗方随意。
- [ ] 执行一次对抗检定。
- [ ] 检查输入框最新 `<meta:检定结果>`：
- [ ] 必须包含 `千早爱音`。
- [ ] 必须不包含 `千早爱音,爱音`（逗号别名串不应原样出现）。

## 2) 效果文案去重与换行（高优先级）

- [ ] 使用会触发二级/三级效果的检定（例如 SAN 检定预设）。
- [ ] 检查 `<meta:检定结果>` 中主检定叙事与效果行之间没有多余空白行（不应连续空两行）。
- [ ] 检查未重复出现“命中【X】分支，分支判定式 ...”这类与主检定语句语义重复的行。
- [ ] 若配置了复杂条件（例如 `智力>90 && SAN变化>10`），效果行仍能表达触发结果（成立/不成立或后果文本）。

## 3) PartC 互通：UI 与 API 历史共享（高优先级）

- [ ] 先通过 UI 做一次对抗检定。
- [ ] Console 执行 `AcuDice.getHistory({ limit: 5 })`，应看到刚才那条对抗记录。
- [ ] 再用 Console 执行一次 `AcuDice.check({ attribute: '力量', targetValue: 50 })`。
- [ ] 打开历史面板，应看到这条 API 检定记录。
- [ ] 再执行 `AcuDice.getHistory({ limit: 5 })`，应同时看到 UI 对抗 + API 检定，且按时间倒序。

## 4) 非回归冒烟（中优先级）

- [ ] `AcuDice.roll('2d6')` 返回结构正常。
- [ ] `AcuDice.roll('3+5')` 返回 `total=8`。
- [ ] `AcuDice.roll('invalid@@dice')` 抛错。
- [ ] `AcuDice.listPresets()`、`AcuDice.getPresetSummary('__builtin_default__')` 正常。

## 通过标准

- 1/2/3 三组全部通过，视为本轮修复有效。
- 若仅第 2 组在个别预设下异常，请记录触发预设 ID、输出全文和重现步骤。
