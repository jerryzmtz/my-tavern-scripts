## 总体判断

当前 UI 功能完整，教程覆盖广，入口多，已经能支持复杂跑团工作流。主要问题不是功能缺失，而是视觉和交互语言过于分散：同类按钮、提示、弹窗、动效、强调方式在不同界面域里不完全一致，且大量模板字符串内联样式让后续统一成本变高。

默认方向应收敛为稳重工具感：保留主题能力，先让 `native`、`modern`、`dark` 在高频路径上稳定、清晰、可扫读。

本次重审的核心目标是统一美学系统。后续每批不能只修当前屏幕的局部观感，而要把当前界面域迁移到同一套组件、间距、层级、状态和动效规则里。专题界面可以保留内容气质，但不能重造控件语言。

## 优先问题

### P1：组件语言不统一

同类按钮、图标按钮、提示、弹窗、菜单和卡片在不同界面域里出现不同形态。后续第 1 批应先建立基础组件规则，避免每个界面单独修。

验收标准：

- 主按钮、次按钮、图标按钮、危险按钮、禁用按钮状态清晰。
- focus、hover、active、disabled、error、warning、success 的变量和样式可复用。
- `native`、`modern`、`dark` 三个主题下对比度明确。
- 同一语义动作在仪表盘、表格、设置、投骰、MVU、背包、抽卡中使用同一种视觉表达。

### P1：自动检测明确命中的禁用样式

厚单侧强调线、弹性动效、布局属性动画都已经被检测到。它们是第 1 批的低争议改造项。

验收标准：

- 当前界面域内不再新增 `border-left` 或 `border-right` 大于 1px 的强调卡片。
- 弹性曲线替换为平滑 ease-out 曲线或移除。
- 当前界面域避免动画 `width`、`height`、`max-height`、`padding`、`margin`。

### P1：高频路径认知负担偏高

仪表盘、表格头部和投骰面板都承担大量操作。当前已经能用，但用户需要在很多小入口之间辨认主要动作。

验收标准：

- 每个界面域明确一个主要动作，其余动作按频率和风险降级。
- 图标按钮保留清晰 title 或 aria-label。
- 移动端操作目标不小于当前触屏可用标准，文字不重叠。

### P2：状态与反馈不成体系

加载、保存、刷新、失败、空状态、危险确认和撤销反馈散落在不同实现中。

验收标准：

- 当前界面域至少覆盖空状态、错误状态、保存成功、取消、关闭和危险确认。
- 错误文案告诉用户发生了什么，以及下一步能做什么。

### P2：主题个性与产品可用性冲突

强个性主题保留价值，但默认体验需要先稳定。后续应把主题分为基础验收主题和个性主题，避免所有主题同时阻塞主路径改造。

验收标准：

- 第 1 批先验证 `native`、`modern`、`dark`。
- 个性主题不得破坏按钮文字、active 文本、输入文本和 warning/error 文本的可读性。

## 分批验收顺序

1. 第 1 批：视觉系统与组件基础。先处理自动检测命中的禁用样式，并统一基础状态。
2. 第 2 批：主路径壳层。处理导航、面板头部、仪表盘层级和移动端误触，同时把主路径迁移到统一组件语言。
3. 第 3 批：数据浏览与编辑。处理表格卡片、菜单、收藏、变更审核、MVU，并统一编辑和状态反馈。
4. 第 4 批：投骰与规则工作流。处理普通检定、对抗、历史、规则预设，并统一表单和主要动作表达。
5. 第 5 批：专题界面。背包、抽卡、地图、关系图、头像、教程遮罩逐个验收，保留专题气质但服从统一控件规则。
