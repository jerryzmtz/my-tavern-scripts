# 如何修改骰子表格而不丢失脚本功能

## 总原则

- 最稳妥做法：保留默认表名和关键列名，只增删非关键列。
- 如果大幅改名，请在“全能设置 → 自定义仪表盘”里配置表名和列名映射。
- 属性、关系、资源这类键值对，推荐统一写成 `名称:数值; 名称:数值`  `角色名:关系词; 角色名:关系词`这种格式。
- 内置数据验证和部分下拉/赠与功能仍依赖默认表名；想完全保留功能，建议保留 `主角信息`、`重要角色表`、`物品表`、`装备表` 等默认名。

## 功能与模板要求

| 功能 | 模板需要满足的条件 |
| --- | --- |
| 可视化表格编辑 | 任意表都能显示。建议第 1 列为 `row_id`，第 2 列为名称/标题，方便卡片标题、跳转、锁定和差异对比。 |
| 行动选项按钮 | 表名含地点/人物/角色/物品/装备/技能/任务/势力等关键词时会有默认按钮。若有列名包含 `交互`，内容可写 `交谈,观察,离开`，会追加成按钮。 |
| 选项表 | 表名包含 `选项`。除第 1 列外的非空单元格会变成可点击选项。默认验证期望选项表最多 1 行。 |
| 普通检定/对抗检定 | 需要能找到角色名列和属性列。主角表名建议含 `主角`；NPC 表建议使用 `重要角色表` 或 `重要人物表`。属性列名需包含 `属性`，推荐 `基础属性`、`特有属性`。 |
| 属性生成/清空/自动改值 | 同上。生成属性会写入 `基础属性`；若存在 `特有属性` 或 `特别属性` 列，会把特有属性分开写入。 |
| 自定义属性规则 | 任意表格模板 note 中只要有闭合的 `<属性规则>...</属性规则>`，切换属性规则时就会同步更新该段。推荐放在主角表和重要角色表的 note 中。 |
| 检定建议表 | 表名包含 `检定建议`。需要 `展示文本` 列和 `骰子命令` 列；找不到时默认用第 2、3 列。note 中需要 `<检定规则>...</检定规则>` 才能跟随检定预设同步。 |
| 人物关系图 | 默认直接打开关系图时，角色表需要名称列，并且有列名包含 `人际关系`。值推荐 `御坂美琴:同学; 白井黑子:朋友`，也支持 `御坂美琴(同学)`。可选 `在场状态` 列用于筛选。 |
| 主角关系 | 主角表若要参与关系图，表名需为 `主角信息`，并有 `人际关系` 列。 |
| 仪表盘 | 默认识别下方“仪表盘默认识别规则”。若改成恋爱对象表、装扮表、备忘录等，请用自定义仪表盘预设映射。 |
| 地图模块 | 需要地点表和地点名列；推荐保留 `世界地图点.详细地点`。地图元素表名需含 `地图元素`/`元素表`/`地图要素`/`机关`/`线索`，并有 `元素名称`、`元素类型`、`所在地点` 等列。所有“所在地点”应填写世界地图点里的详细地点名。 |
| 物品栏 | 需要能被识别为背包/物品表。默认 `物品表` 最完整；关键列为 `物品名称`、`类型`、`数量`、`品质`、`描述`。 |
| 骰子商店 | 奖励会写入物品表或装备表。物品表至少要能识别名称、类型、数量；装备表至少要能识别名称、类型、状态。缺少 `数量` 会影响堆叠，缺少 `品质/描述` 会影响筛选和详情。 |
| 装备展示 | 装备表关键列为 `装备名称`、`类型`、`部位`、`状态`、`品质`、`描述`。状态写 `已装备`/`装备中`/`是` 等会被仪表盘识别为当前装备。 |
| 头像管理 | 依赖角色名称。角色表名和名称列能被仪表盘或关系图识别时，头像会跟随角色名显示。 |
| 数据验证 | 内置验证主要针对默认表名和默认列名，例如 `主角信息.基础属性`、`重要人物表.人际关系`、`世界地图点.详细地点`。重命名后建议同步修改验证规则。 |

## 仪表盘默认识别规则

| 仪表盘区域 | 默认识别的表名关键词 | 关键列关键词 |
| --- | --- | --- |
| 全局状态 | `全局数据表`、`全局数据`、`全局` | `当前详细地点`、`当前次要地区`、`当前时间` |
| 主角 | `主角信息`、`主角`、`玩家`、`角色信息`、`user`、`<user>` | `姓名/名称`、`状态`、`位置/所在地`、`基础属性/属性`、`资源/金钱` |
| 地点 | `世界地图点`、`地图点`、`地图`、`地点`、`场景`、`区域` | `详细地点/地点名/名称`、`环境描述/描述` |
| NPC/角色 | `重要角色表`、`重要人物表`、`NPC`、`人物`、`角色`、`伙伴`、`队友` | `姓名/名称`、`状态`、`位置/所在地点`、`在场状态` |
| 任务 | `任务表`、`任务`、`事项`、`目标`、`待办`、`委托` | `名称`、`类型`、`进度`、`状态`、`优先级` |
| 物品 | `物品`、`背包`、`道具`、`库存`、`持有物品表` | `物品名称/名称`、`类型`、`数量` |
| 装备 | `装备表`、`装备`、`武器`、`防具`、`法宝` | `装备名称/名称`、`类型`、`部位/位置`、`状态` |

## 推荐保留的默认核心表

- `全局数据表`：至少保留 `当前详细地点`、`当前次要地区`、`当前时间`。
- `主角信息`：至少保留 `姓名`、`自身状态`、`所在地点`、`基础属性`、`特有属性`、`资源数据`。
- `重要角色表`：至少保留 `姓名`、`基础属性`、`特有属性`、`所在地点`、`在场状态`、`人际关系`、`交互选项`。
- `世界地图点`：至少保留 `详细地点`、`次要地区`、`环境描述`。
- `地图元素表`：至少保留 `元素名称`、`元素类型`、`所在地点`、`元素描述`、`状态`、`交互选项`。
- `物品表`：至少保留 `物品名称`、`类型`、`数量`、`品质`、`描述`。
- `装备表`：至少保留 `装备名称`、`类型`、`品质`、`状态`、`描述`，如需堆叠装备可加 `数量`。
- `任务表`：至少保留 `名称`、`类型`、`进度`、`状态`，如需仪表盘排序可保留 `优先级`。
- `检定建议表`：保留 `展示文本`、`骰子命令`，并在 note 中保留 `<检定规则>...</检定规则>`。

## 常用格式

```text
属性：力量:55; 敏捷:40; 侦查:70
资源：金币:250; 行动点:3
关系：御坂美琴:同学; 白井黑子:朋友,队友
交互选项：交谈,观察,赠送
骰子命令：检定 <user> 侦查 难度=困难
骰子命令：对抗 <user> 心理学 vs 守夜人 话术
骰子命令：无 / 必成 / 必败
```

## 改名时的简短建议

小改表头可以，只要保留关键词；大改表名时，请优先配置自定义仪表盘。若你希望“下拉角色、赠与、内置验证、主角关系图”也完全无损，尽量不要改 `主角信息`、`重要角色表`、`物品表`、`装备表` 这些核心表名。
